# Output Templates

## PRE-VIBE-SPEC.MD

- Session meta
- Raw user input
- Normalized goal
- Scenario and audience
- Budget and compression mode
- Scope
- Requirements
- Acceptance criteria
- Context summary
- Tutorial suggestions
- Assumptions and unknowns
- Risks
- Verification plan
- First prompt pointer

## INIT-AGENTS.MD

Only include durable project or workflow guidance. Do not include one-off task requirements. Combine with global Codex AGENTS by summarizing global constraints and adding non-conflicting project-level rules.

## FIRST-PROMPT.MD

Keep this short, direct, and execution-ready. It should point to the spec file instead of copying the full spec. It is the only default context injected after `/clear`.
